基于混合算法的互联网访问用户身份认证方法 
王家峰 
（安徽广播影视职业技术学院，安徽 合肥 230011） 
 
摘要：提出基于混合算法的互联网访问用户身份认证方法，结合域内联盟链和跨域联盟链，建立教育区块链身份
认证框架。在认证架构中设置固定的权威节点后，结合无证书签名以及哈希函数，完成机构节点、用户节点的合
理注册。建立基于混合算法的安全认证机制，首先应用国密算法中的 SM4 算法（SM4 分组密码算法）产生一个初
始密钥；然后利用 SM2 算法（SM2 椭圆曲线公钥密码算法）对密钥数据进一步处理，获取可以用于身份认证的随
机密钥；最后，以云计算为核心的认证服务器的作用下，对互联网访问用户身份进行阶段认证。实验结果表明，
在并发用户数量为 300 时，所提方法的互联网访问用户身份认证响应时间仅为 2.85
 s，满足实时性要求。 
关键词：混合算法；互联网；访问控制；身份认证；密钥；区块链

互联网技术得到普遍应用，企业网、校园网、教育科研网等不断涌现
[1]。随着网络建设进程的加快，
网络安全问题呈现在人们眼前，为了打造功能齐全、安全可靠的网络业务系统，需要将身份认证技术和机
制引入到互联网业务系统中
[2]。针对访问用户身份进行统一认证，可以最大程度避免网络风险。 
当前，已经应用新型算法和技术设计不同类型的互联网访问用户身份认证方法。黄敏敏等
[3]采用主从
多链结构，搭建三层身份认证架构，并引入边缘计算算法，构建分布式安全可信认证模型。在边缘计算过
程中，利用智能合约详细划分用户访问权限，并结合椭圆曲线加密算法进行签名认证。但是，该认证算法
安全性较差。赵宗渠等
[4]以提升网络访问安全性为核心，运用双线性映射算法、身份匿签密算法，设计一
种身份隐藏认证方案。通过认证方案进行会话密钥协商，实现网络访问用户的双重身份认证。但是，该认
证方法计算开销较大。于光华等
[5]提出在云服务器上连接数个物联网，构造云平台物联网，作为远程身份
认证的基础。依托于轻量级多因子智能卡构造原理，设计包含预计算、注册、身份认证、智能卡撤销 4 个
主要环节的自动化用户身份认证方案。但是，该认证方案的应用拓展性较差。 
考虑当前身份认证研究成果无法满足互联网安全访问要求，本文提出一种基于混合算法的新型互联网
访问用户身份认证方法，将国密算法中的 SM2 算法和 SM4 算法结合起来，高效获取更加安全的认证密钥，
基于密钥得出准确的身份认证结果。 
1  应用混合算法设计互联网访问用户身份认证方法 
1.1  构建教育区块链身份认证框架 
为了保证身份认证工作的顺利进行，本文采用区块链思想，构建包括域内联盟链、跨域联盟链两种主
要结构的教育区块链认证框架，如图 1 所示。认证架构中，域内联盟链主要负责对象是目标区域中包含的
机构节点
[6]，对所有访问这些互联网机构的用户进行身份认证。跨域联盟链则是在本区域内用户访问其他
区域机构时，可以进行身份认证。 
由图 1 可知，基于教育区块链的认证模型架构中，主要包括 3 类节点，分别是用户节点、机构节点和
权威节点。其中，权威节点是固定不变的，每一个权威节点都代表一个认证服务器，负责为用户颁发身份 
证明。机构节点是指互联网中待访问节点，用户节点是指发出访问请求的用户。

1.3  建立基于混合算法的安全认证机制 
所有注册成功的节点都成为区块链上的合法用户，
可以与其他节点之间进行通讯。在互联网访问用户身份
认证过程中，在权威节点的辅助下，实现域内机构节点
和用户节点的双向身份认证
[9]。为了简化身份认证环节，
采用国密算法中的 SM2 算法和 SM4 算法，建立基于混
合算法的安全认证机制。 
实际操作过程中，将用户身份认证分为 3 个环节，
分别是普通节点与机构节点通信、权威节点与机构节点通信以及普通节点与权威节点相互认证。设置每个
权威节点对应一个认证服务器，由用户节点向权威节点发送身份认证请求，认证服务器确定用户请求服务
合法时，运用混合算法生成公钥和加密会话密钥作为共享密钥
[10]，作为后续互联网访问身份认证的依据。在
混合算法的作用下，加密和解密流程如图 4 所示。

身份认证过程中，通过机构节点的帮助将该密钥传递给权威节点，在验证密钥的正确性后确定用户身份是
否安全。

A hybrid algorithm based authentication method for internet access users 
WANG Jia-feng 
(Anhui Broadcasting Movie and Television College, Anhui Hefei 230011, China) 
 
Abstract：A hybrid algorithm based internet access user identity authentication method is proposed. Establish an 
education blockchain identity authentication framework by combining intra domain and cross domain alliance chains. 
After setting a fixed authoritative node in the authentication architecture, combined with certificateless signature and 
hash function, complete the reasonable registration of institutional nodes and user nodes. Establish a security 
authentication mechanism based on a hybrid algorithm. First, use the SM4 algorithm (SM4 Block cipher algorithm) in 
the national security algorithm to generate an initial key, and use the SM2 algorithm (SM2 elliptic curve public key 
cryptography algorithm) to further process the key data to obtain a random key that can be used for identity 
authentication. Finally, under the role of an authentication server centered on cloud computing, stage authentication is 
performed on the identity of internet access users. The experimental results show that when the number of concurrent 
users is 300, the proposed method only has a response time of 2.85 seconds for internet access user authentication, 
meeting the real-time requirements. 
Key words：hybrid algorithm；internet；access control；identity authentication；key；blockchain
