## 五、引用格式指南（v4适用）

### 中文期刊
```
作者. 题名[J]. 刊名, 年份, 卷(期): 页码.
示例：[36] 王卓. 典型区块链技术的发展综述[J]. 电子技术, 2025, 54(01): 114-115.
```

### 国际会议
```
作者. 题名[C]. 会议名, 年份.
示例：[40] Castro M, Liskov B. Practical Byzantine Fault Tolerance[C]. OSDI, 1999.
```

### 学位论文
```
作者. 题名[D]. 学位授予单位, 年份.
示例：[44] 张瑞星. 基于区块链技术的食品溯源平台关键技术研究[D]. 电子科技大学, 2021.
```

### 国家标准
```
发布单位. 标准号 标准名称[S]. 出版社, 年份.
示例：[53] 国家密码管理局. SM2椭圆曲线公钥密码算法: GB/T 32918.1-2016[S]. 中国标准出版社, 2016.
```

### 在文中引用方式（v4规范）

#### ✗ 错误（v3风格）：
```
作者等[n]针对…进行了研究，发现…
例：Castro等[40]针对拜占庭问题进行了深入研究，提出了PBFT算法。
```

#### ✓ 正确（v4风格）：
```
…相关技术原理[n]。
或
…的定义[n]。
或
…已被证实[n]。
例：拜占庭容错问题的解决方案由PBFT算法提供[40]。
```

## 六、后续使用建议

### 文章写作建议
1. **Chapter 2 初稿**：确保每个技术概念都对应至少一篇基础论文引用
2. **引用检查**：避免在理论基础中出现"作者等…针对…进行了研究"的表述
3. **密度控制**：理想密度 ~200字/篇，保持自然流畅
4. **标准优先**：密码学和农产品追溯部分优先使用 GB/T 标准

### 引用编辑工具建议
- 使用 BibTeX 或 Mendeley 管理此60篇文献
- 配置引用格式为 GB/T 7714（中文期刊标准）
- 定期交叉检查文中引用号与此清单的一致性

### 版本管理
- v4 为最终定版方案，后续仅做微调不改编号
- 如需替换某篇文献，改为 v4.1、v4.2 等补丁版本
- 保留 v3 版本作为对比参考
